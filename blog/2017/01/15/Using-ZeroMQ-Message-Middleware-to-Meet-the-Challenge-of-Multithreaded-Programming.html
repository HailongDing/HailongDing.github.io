Using ZeroMQ Message Middleware to Meet the Challenge of Multithreaded Programming

Posted on January 15, 2017 ()

Categories: multithreaded programming  message middleware 

As we all know, it’s not easy to write multithreaded programs. On one hand, we should strive to use the synchronization mechanism carefully; on the other hand, we should avoid jeopardizing the program’s performance. In recent years, lock-free programming is introduced as a way to safely share changing data between multiple threads without the cost of acquiring and releasing locks. This sounds like a panacea, but it is complex and subtle, and sometimes doesn’t give the benefits that it promises.

Fortunately, we have another choice. We can use the messaging middleware to help us get ride of such burden, and achieve both convenience and high performance goals. To demonstrate this technology, we will use traditional multithreaded skills compared with the famous messaging middleware ZeroMQ to investigate the usability and performance.
