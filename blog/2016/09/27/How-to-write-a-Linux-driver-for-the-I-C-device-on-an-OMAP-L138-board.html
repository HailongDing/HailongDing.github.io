How to write a Linux driver for the I²C device on an OMAP-L138 board

Posted on September 27, 2016 ()

Categories: linux driver  OMAP-L138 

Although the OMAP-L138 module offers three RS232 serial ports, customers often prefer to use RS485 for industrial device communication, due to its robustness for long-distance networking in noisy environment. In one of our products, we used an NXP SC16IS752 chip in the I²C interface for this purpose. The SC16IS752 is an I²C-bus/SPI bus interface to a dual-channel high performance UART. I will show you how to write a Linux driver for this device.

First of all, the very important hardware background we should know is the I²C address used in this application according to its connection with the board - that’s 0x4D.
